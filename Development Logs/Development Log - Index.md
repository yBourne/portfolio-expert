# Development Log - Index

> Investment Assistant 项目开发进度日志。每完成一个明确功能点 / 文档里程碑，都必须新增一条日志，并分配唯一版本号。

## 日志规则

- 日志目录：`Projects/Investment Assistant/Development Logs/`
- 版本号格式：`IA-DEV-YYYYMMDD-NNN`
  - `IA` = Investment Assistant
  - `DEV` = development log
  - `YYYYMMDD` = 记录日期
  - `NNN` = 当日递增序号，从 `001` 开始
- 每个功能点 / 开发节点单独一条日志。
- 每条日志至少包含：版本号、日期、类型、状态、目标、完成内容、涉及文件、验证结果、下一步。
- 代码开发必须记录实际测试命令和结果。
- 文档 / 规则更新必须记录被更新的 Obsidian 文档和是否同步索引。
