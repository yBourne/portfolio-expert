# Knowledge Base Sources

> 投资知识库资料来源规划。用于 NotebookLM、Obsidian 和 Knowledge Agent 的资料管理。

## 1. 定位

- Obsidian：长期维护结构化笔记、规则、复盘和项目文档。
- NotebookLM：用于阅读大量源材料、问答、摘要和带引用的理解。
- Knowledge Agent：从资料中提取可用于决策和复盘的原则，而不是直接给交易信号。

## 4. 知识库使用原则

- 不把大师语录当作交易信号。
- 只提取长期有效的原则。
- 每条原则必须能对应到具体行为：买入前检查、卖出前检查、仓位限制或复盘问题。
- 和个人规则冲突时，需要记录冲突，不自动覆盖规则。
