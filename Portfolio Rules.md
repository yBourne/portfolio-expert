# Portfolio Rules

> 可执行的组合约束和风险规则。相比 [[Investment Constitution v0.1]]，本文件更结构化，后续可映射到规则引擎。

## 1. 规则优先级

1. 风险控制规则优先于收益优化。
2. 投资宪法优先于单次模型建议。
3. 硬性仓位上限优先于个股评分。
4. 人工确认优先于自动化执行。

## 2. 默认仓位约束（Reviewed）

- 指数 / ETF 目标仓位：30% - 50%。
- 个股总仓位上限：70%。
- 现金 / 类现金最低比例：0%。
- 单只个股上限：25%。
- 高风险单只个股上限：10%。
- 单一行业 / 主题上限：40%。
- 单次调仓总变动上限：组合总资产的 15%。

> 已 Review：以上比例按当前真实偏好执行；后续可随风险承受能力变化再调整。

## 3. 再平衡触发条件

触发组合诊断：

- 任一单票超过上限。
- 任一行业 / 主题超过上限。
- 指数 / ETF 仓位低于 30% 或高于 50%。
- 现金比例出现负数或数据异常。
- 市场状态从 neutral 切换为 risk-off。
- 新股票评分显著高于当前最低质量持仓。
- 个股基本面评分连续下降。
- 单周组合回撤超过预设阈值。

## 4. 个股评分与动作映射

- 高质量 + 估值合理 + 风险可控：允许纳入候选或加仓。
- 高质量 + 估值偏贵：观察，不追高。
- 质量一般 + 风险上升：降低目标仓位。
- 基本面恶化 + 逻辑被证伪：进入退出候选。
- 信息不足：不得给出强买入建议。

## 5. 市场状态下的仓位规则

### Risk-on

- 可维持或适度提高个股仓位。
- 不允许突破单票和行业上限。
- 优先选择高质量、流动性好、风险收益比明确的标的。

### Neutral

- 优先保持目标仓位。
- 减少频繁调仓。
- 只对明显超限或明显低质量持仓做调整。

### Risk-off

- 提高指数 / ETF / 现金权重。
- 降低高 beta、高估值、高不确定性个股权重。
- 新增个股必须更严格审核。

### High-volatility

- 降低换手。
- 避免被短期波动驱动。
- 所有大额操作进入冷静期。

## 6. 交易前检查清单

每次交易前检查：

- 是否符合 [[Investment Constitution v0.1]]？
- 是否突破单票上限？
- 是否突破行业 / 主题上限？
- 是否降低组合分散度？
- 是否有明确理由和退出条件？
- 是否由单一新闻或情绪触发？
- 是否已记录到 [[Trade Log Template]]？

## 7. 交易后复盘规则

每次交易后记录：

- 实际操作。
- 操作理由。
- 当时市场状态。
- 是否符合规则。
- 1 周 / 1 月后复盘结论。
- 如果重来一次，是否还会做同样决策。
